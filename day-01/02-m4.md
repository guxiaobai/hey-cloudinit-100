# 最推荐的方案（M4）

```
qemu-system-aarch64 \
    -machine virt,accel=hvf \
    -cpu host \
    -m 1024 \
    -nographic \
    -bios /opt/homebrew/share/qemu/edk2-aarch64-code.fd \
    -drive file=noble-server-cloudimg-arm64.img,if=virtio,format=qcow2 \
    -netdev user,id=net0 \
    -device virtio-net-pci,netdev=net0 \
    -smbios type=1,serial=ds='nocloud;s=http://10.0.2.2:8000/'
```

# 一、先下载 ARM64 镜像

```bash
wcurl https://cloud-images.ubuntu.com/noble/current/noble-server-cloudimg-arm64.img
```

# 三、关键变化说明

------

## 1. 改成 ARM 虚拟机

原来：

```
qemu-system-x86_64
```

改成：

```
qemu-system-aarch64
```

因为 M4 是 Apple M4 ARM 架构。

------

## 2. KVM 改成 HVF

原来：

```
-machine accel=kvm:tcg
```

改成：

```
-machine virt,accel=hvf
```

macOS 用的是 Hypervisor.framework。

------

## 3. 磁盘改为 virtio

原来：

```
-hda noble-server-cloudimg-amd64.img
```

改成：

```
-drive file=noble-server-cloudimg-arm64.img,if=virtio,format=qcow2
```

这样性能更好。

------

## 4. 网络参数改新版

原来：

```
-net nic
-net user
```

改成：

```
-netdev user,id=net0
-device virtio-net-pci,netdev=net0
```

因为旧的 `-net` 参数已经不推荐使用。

------

## 五、如果启动报错，需要加 BIOS

某些 ARM cloud image 需要 UEFI：

```
-bios /opt/homebrew/share/qemu/edk2-aarch64-code.fd
```
